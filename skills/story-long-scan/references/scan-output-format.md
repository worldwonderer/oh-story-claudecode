## browser-cdp 采集流程

### 通用流程

```
1. 启动 browser-cdp，打开目标榜单页面
2. 等待页面加载完成（检查列表元素出现）
3. 提取列表页数据（排名、书名、作者、字数、推荐/在读等）
4. 判断是否需要翻页：
   - 起点榜单通常单页展示完整（50-100条）
   - 番茄按题材逐页采集，每页 cap≈20
5. 逐条进入详情页获取补充数据：
   - 标签/标签区
   - 简介全文
   - 最新更新信息
6. 汇总写入 Markdown 文件
```

### 起点采集要点

- 榜单页直接包含：排名、书名、作者、题材、字数、推荐数、签约状态、免费/VIP
- 详情页需要提取：标签、最新更新、简介
- 三江榜按周分组，需识别周标题边界
- 简介可能包含换行和格式，原文保留

### 番茄采集要点

- 按题材逐个采集，每个题材一个 URL
- 题材列表可从榜单页面的分类导航提取
- 在读数是番茄特色指标（如"34.1万"）
- 单页展示上限约20本，需要翻页或多次请求获取更多
- 简介较长时需要展开

### 晋江采集要点

- 页面结构较老，可能需要特殊处理
- 收藏数、营养液是核心差异指标
- 需要注意登录态（部分数据需登录）

## 文件命名规范

```
{平台}{榜单名称}_{日期}.md
```

示例：
- `起点新人签约新书榜_20260425.md`
- `起点三江近四周_20260425.md`
- `番茄男频新书榜_全题材_20260425.md`
- `晋江金榜_20260425.md`

## 批量采集模式

用户说"全榜扫描"时：

| 平台 | 默认采集组合 |
|------|-------------|
| 起点 | 新人签约新书榜 + 三江推荐榜 + 月票榜前20 |
| 番茄 | 男频新书榜全题材 + 女频新书榜全题材 |
| 晋江 | 金榜 + 新文榜 |
| 全平台 | 起点默认组合 + 番茄默认组合 |

每组采集完成后自动保存文件，再进入下一组。
