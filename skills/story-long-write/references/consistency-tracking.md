# 连续性追踪与事件冷却

## 热度追踪系统

用于管理长篇网文中的人物、支线、伏笔的出场频率，防止遗忘或过度使用。

### 热度状态

| 状态 | 定义 | 说明 |
|------|------|------|
| hot | 当前驱动冲突的核心元素 | 正在推动剧情 |
| warm | 近期活跃的元素 | 读者还记得 |
| cold | 超过安全线未触及 | 有被遗忘的风险 |
| archived | 已完结/有意关闭的元素 | 不需要再追踪 |

### 有效触碰定义

以下算有效触碰（维持热度）：
- 直接推进该线索
- 施加压力提醒读者它的存在
- 改变该元素的关系状态
- 产生实际后果
- 交代合理的休眠原因

**不算有效触碰：**
- 纯粹提个名字（走过场）
- 空头回调（没有新信息的回忆）
- 为了满足规则随机提及

### 回顾阈值

| 元素类型 | 触及间隔 | 说明 |
|----------|----------|------|
| 核心角色 | 3-5 章 | 超过 5 章不出现需要交代原因 |
| 主要支线 | 4-6 章 | 超过 6 章需要至少一个 reminder |
| 活跃伏笔 | 2 次错过机会 | 有合适场景却没有触碰 |
| 不稳定关系 | 2 次出场 | 关系正在变化中，频繁追踪 |
| 世界设定 | 长时间缺席后 | 补充提醒读者规则 |

### 每章自检问题

写完每章后问自己：
1. 当前 hot 的元素是什么？
2. 有没有 cold 了但该 warm 的？
3. 哪些可以合理保持休眠？
4. 哪条线索的回归能加深压力？

### 失败信号

- 读者问"那个谁去哪了？"
- 重要的线索到结尾才突然冒出来
- cold 的铺垫突然变成 hot 的回报（没有预热）
- 关系突然剧变但没有前期铺垫

## 反速解机制

防止 AI 倾向于过早解决冲突的核心规则。

### 核心规则

1. **非大结局章节禁止解决核心冲突**
   - 核心冲突由大纲定义（主线矛盾）
   - 每章必须新增至少一个未解决的次级问题

2. **章末 200 字必须包含悬念元素**
   - 提问型："她到底在隐瞒什么？"
   - 危机型："门在响，但没有人应该知道她在这里。"
   - 反转型："他打开盒子。里面不是他期待的东西。"

3. **允许解决的范围**
   - 只能解决次级冲突
   - 主线可以推进，但不能关闭
   - 局部胜利必须伴随新的代价或风险

## 事件冷却矩阵

防止同一类型事件连续出现导致节奏单调。

### 5 类事件与冷却期

| 事件类型 | 情绪效果 | 冷却（章） | 说明 |
|----------|----------|-----------|------|
| conflict_thrill | 紧张→满足 | 2 | 大冲突/打斗/对峙 |
| bond_deepening | 温暖→感动 | 1 | 关系深化/暖心互动 |
| faction_building | 成就→掌控 | 2 | 建立势力/获得资源 |
| world_painting | 沉浸→好奇 | 3 | 世界观展开/新设定 |
| tension_escalation | 不安→期待 | 2 | 压力升级/危机逼近 |

### 规则

- 冷却期内该类型不能作为主 beat
- conflict_thrill 最多连续 2 章
- 每 5 章必须包含 bond_deepening 或 world_painting（不能全是打打杀杀）
- 非冲突事件中要求埋入微伏笔（路人→未来关键角色，商品→未来剧情道具）

### 使用方法

每写完一章，更新事件状态表：

```
最近事件记录：
- 第X章：conflict_thrill（冷却到第X+2章）
- 第X+1章：bond_deepening（冷却到第X+2章）
→ 第X+2章可用：bond_deepening(冷却结束)、tension_escalation(冷却结束)
→ 第X+2章禁用：conflict_thrill(冷却中)
```

## 过渡章节管理

### 单元故事过渡期目标感消失问题

数据显示，使用一级结构时，单元故事结束后的过渡章节跟读会下降。
**原因**：单元故事结束后没有钩子留住读者。

**解决方案**：
- 结束当前剧情之前，先把下一个目标拉出来
- 当前剧情结束之后主角自然有事可做
- 过渡章节必须维持至少一条活跃的期待线

### 换地图期待感延续

换地图是期待感最容易断裂的时刻，必须提前铺设：

| 延续方式 | 做法 | 效果 |
|----------|------|------|
| 复仇线 | 未完成的目标跨地图持续 | 成为新地图的大高潮目标 |
| 旧日关系线 | 老角色在新地图出现 | 关系延续，读者不觉得断裂 |
| 信息差 | 某方以为某事，实际不是 | 期待"真相揭露"的反应 |
| 提前铺垫 | 换地图前让新地图角色/传说与主角接触 | 换地图前就有期待感 |

### 换地图时机判断

- 主角在当前地图地位达到顶点，关系网潜力耗尽
- 无法再设计新的装逼场景
- 平级换地图要谨慎：保持切换节奏一致（如开头三个小区才换城市，后面也保持）

### 日常路线扩展关系网

用主角日常路线（上学/归还装备/逛集市）作为新故事弧线的桥梁。每个日常接触点可引出新角色，该角色自带关系网。

## 金手指体系生命周期

防止金手指僵化导致后期失去拉期待能力的核心规则。

### 四阶段递进

| 阶段 | 特征 | 操作要点 |
|------|------|---------|
| 基础 | 金手指首次展示，功能单一 | 明确核心作用，建立读者认知 |
| 发展 | 功能拓展，使用场景增多 | 增加新的使用方式，但核心作用不变 |
| 成熟 | 功能完善，读者已熟悉运作模式 | 与世界观深度结合，可与其他系统联动 |
| 升华 | 超越原有框架，格局质变 | 有足够伏笔支撑，形成周期性循环爆发 |

**关键规则**：
- 金手指重心可转移，但必须有足够伏笔
- 可部分淡化，不能完全抛弃
- 呈现力度应随阶段递增，而非递减

### 统合性金手指消弭换地图割裂

用跨越地图的统合性金手指锚定主角团，串联各个区域，避免游记式写法的分散弊病。核心做法：设计一个能跨越空间限制的金手指机制（如聊天群、组织、特殊联系），让换地图时人际关系不断裂。

## 矛盾网设计

防止冲突碎片化、缺乏纵深的核心规则。

### 设计流程

1. 设定各势力的最终目的
2. 找到势力间的终极矛盾核心
3. 从终极矛盾向前推导各势力的行为方式
4. 为每个阶段设计具体的矛盾表现

### 并行矛盾线管理

- 同一时刻保持2-3条矛盾线同时运行
- 矛盾线之间要有关联（因果、利益冲突、信息差）
- 每次解决一个矛盾，必须激活或加深另一个矛盾
- 主角获得收益必须损害某方利益，由此自然接引下一阶段剧情

### 矛盾层级

| 层级 | 范围 | 说明 |
|------|------|------|
| 章级 | 2-3章 | 小冲突，服务于当前单元 |
| 卷级 | 一卷 | 本卷核心矛盾，卷末解决 |
| 书级 | 全书 | 终极矛盾，大结局解决 |

卷与卷之间的矛盾需要服务于全书矛盾，可以是关联的也可以独立，但都在丰富和推进全书核心矛盾。

## 期待感管理系统

防止"断期待"导致追读血崩的核心规则。

### 断期待的常见原因与修复

| 断期待原因 | 表现 | 修复方法 |
|-----------|------|---------|
| 完成目标后无新目标 | 读者失去动力 | 提前给出下一个目标的线索伏笔，不能从零开始 |
| 老角色淡出无替代 | 读者失去情感羁绊 | 提前引入新角色再加入人际网 |
| 换地图时断裂 | 读者失去连续感 | 让老角色随主角进入新地图；换地图前让新地图角色提前接触 |
| 支线太多太杂 | 主线目标模糊 | 砍掉无用支线，保持主线清晰 |
| 长期不满足期待 | 读者耐心耗尽 | 设阶段性小境界、小奖励，逐步引导到大目标 |
| 假满足 | 形式上满足但读者没感受到 | 做好多层次爽点，各阶层配角情绪写到位 |

### "两长一短"期待法则

每个一级结构中至少应包含：
- **1个短期期待**：当前单元的明确目标（只能有一个）
- **1-2个长期期待**：远期目标预告/某个悬念/某个组织/某个人物

短期期待等于限时活动，同一时间最好只有一个。长期期待等于任务列表里的任务，随时可做。

### 持续拉期待的方法

1. 不断在"基底期待"上添加细节，注入新的具体化需求缺口
2. 设置多个核心梗交替运行（如装逼线A + 解密线B + 感情线C）
3. 设计世界观层面的"秘密"和"阴谋"，让剧情犹抱琵琶半遮面

## 升级差异化管理

防止升级后读者感受不到变化、失去升级快感的核心规则。

### 三维差异化

| 维度 | 说明 | 示例 |
|------|------|------|
| 能力差异化 | 每个等级有质变性的新能力 | 凡人纯废物→练气能放法术→筑基能飞行→金丹法天象地 |
| 待遇差异化 | 不同等级获得不同的社会待遇 | 散修→宗师→老祖→传说神话 |
| 人际关系网差异化 | 不同等级面对不同层次的人际关系 | 亲朋倵傲→平等对待→崇拜仰望→远离故人化为传说 |

**关键规则**：如果升级前后在这三个维度上没有明显差异，读者就不会有升级和收获的快感。

## 单元故事嵌套

解决单元故事结束后过渡期目标感消失的问题。

### 嵌套方法

- 在第一个单元故事高潮前，插入第二个故事的期待线
- 完成当前目标前，提前给出下一个目标的线索伏笔
- 完成当前目标后，迅速给出反转或变故，营造新期待

### 日常文写长技巧

- 用对话拉扯维持节奏（非纯灌水，要有信息增量和关系推进）
- 每次小事件串联好感度缓慢增长，避免猛增好感的大事件
- 好感度提升事件要针对不同性格设计专属事件

## 拆书正确方法

### 拆书目的

拆书目的是研究"思路"或优点值得学习的地方。不要纠结于书本身好不好看——你不是受众很正常。

### 带着具体问题拆

- 怎么冲突拉扯情绪？→ 找十段冲突情节分析
- 怎么立对比？→ 专看对比段落，跟对比无关的跳过
- 怎么铺垫期待感？→ 只看铺垫段
- 用图表或文字整理输出，强迫自己思考

### 拆书学习路径

1. **先在市场内寻求可以写的题材**（扫榜）
2. **同题材多阅读几本后的共性研究总结**（找对标书）
3. **找出对标书，研究核心梗/核心情绪链条/结构**
4. **对着具体书做理论总结和思考**

### 对标书拆法

- 对标书可以不止一本
- 自己觉得看得最爽的，也是最容易学会手法的
- 经验不足时别什么都想拆，一上来只围绕一个点攻克
- 比如就要学对比手法，别的什么都不看，只看怎么立对比
- 给段落做标注，感受有和没有的区别
- 注意：别脱离题材拆书，否则越拆脑子越乱

### 素材积累方法

- 定期去番茄/起点扫榜，带着作者思路去看别人怎么写
- 定期做笔记，越积累越明了
- 拆书积累多了会有"应激反应"——及格线以上作品看多了，不合格情节格外扎眼

## 设门槛拉长剧情法

### 核心思路

围绕主角的目标设置阶段性门槛，有效拉长剧情，让主角更有动力。

### 操作方式

| 门槛类型 | 操作 | 效果 |
|----------|------|------|
| 系统门槛 | 首次免费（新手礼包），第二次需资源，后续分普通/超级两档 | 自然消耗资源，持续拉期待 |
| 目标门槛 | 考进顶级院校需气血值+灭杀战绩+比赛排名，逐个提出逐个完成 | 拆分主线为阶段小目标 |
| 收集门槛 | 突破境界需收集多种材料，逐个寻找 | 收集过程本身就是故事 |
| 消耗门槛 | 主角资源太多时提高门槛消耗掉 | 防止资源过剩失去驱动力 |

### 效果

- 拆分主线为阶段小主线，每个阶段都有明确驱动力
- 结合系统门槛+目标门槛，剧情量轻松扩展到50章以上
- 让主角始终处于"差一点就能升级"的期待状态

## 故事卡系统拆解法

### 核心思路

把一段剧情拆成多张"故事卡"，每张卡代表一个情绪节点，排列顺序呈现完整的情绪链条。

### 拆解步骤

1. **写原型故事**：一段话概括整个剧情单元
2. **提取故事线卡**：简化为主角行为链（主角陷入困境→不被看好→金手指→救出→震惊→收获）
3. **拆分故事卡**：按情绪节点拆成独立卡片（如：陷入困境/贵人相助/信息差/递望远镜/获得神器/装逼）
4. **标注期待感**：每张卡的读者期待方向
5. **标注爽点挖掘**：每张卡的爽感释放方向

### 故事卡分析模板

| 维度 | 分析内容 |
|------|----------|
| 卡片名称 | 这张卡的核心事件 |
| 期待感方向 | 读者在这张卡上期待什么 |
| 爽点挖掘方向 | 如何在这张卡上释放爽感 |
| 情绪折线 | 压→抬→压→抬的节奏 |

### 从作者角度倒推剧情设计

| 步骤 | 操作 |
|------|------|
| 设计大高潮 | 确定单元剧情的高潮点（如：杀死Boss、通关副本） |
| 确定解决关键 | 主角靠什么解决（如：信息差制作强力卡牌） |
| 设计信息差逻辑 | 信息差如何制造（如：穿越者知道前世神话） |
| 设定奖励 | 高潮后获得什么 |
| 设定动机 | 主角为什么必须做这件事 |
| 设计爽感层次 | 多层震惊+对比+收获 |

## 同人写作经验

### 为什么推荐新手写同人

- 有固定受众，容易写出信心
- 经典剧情和人物相对固定，降低创作门槛
- 逆推高潮的过程中，自然学会如何正面构建剧情高潮
- 写同人人物不OOC，就是最好的人物描写练习

### 关键经验

- 标题要吸引人：参考贴吧、知乎讨论最焦灼的点，从讨论度最高的剧情切入
- 支线人物压缩：标题对准某个经典人物时，支线人物不超过2章
- 标题偏离核心人物时，反对评论和跳订会明显增加
- 推荐资源要清醒认知目标读者

## 抄书训练法

### 核心思路

抄书不是机械抄写，而是通过手抄+思考+对比的自我训练方法。

### 四种抄书方法

| 方法 | 操作 | 目的 |
|------|------|------|
| 抄前三章 | 手抄对标文前三章→思考行文布局→合上原文用自己的话写→对比差异 | 找出遗漏的信息点、布局差别、用词情绪差异 |
| 抄人设 | 摘录前20章某人物所有描写和剧情→感受人物如何从各维度建立→参考建立自己的人设 | 学习人物渐进式建立方法 |
| 抄印象段落 | 摘抄印象深刻的部分→分析哪些词句调动了什么情感 | 积累情绪调动技巧 |
| 抄结构 | 拆某个不擅长的一级结构为节点→抄作者如何处理每个节点 | 弥补自己的弱项 |

### 操作要点

- 手抄而非打字：手写过程能感知到眼睛看时忽略的信息点
- 抄完必须思考，不过脑子等于白抄
- 不贪多，质量重于数量
- 抄完用自己的话重写，再对比差异——这个环节是开窍的关键
- 新锐作者的畅销文更适合抄，大神的开头不一定适合

## 对标书选择和使用流程

### 正确流程

```
广泛阅读同题材成绩书（用读者状态看）
  ↓ 看到不好看直接弃
确定题材内有哪些方向
  ↓ 找相似性
确定自己要写的方向
  ↓ 
选5-10本参考书（读2-3遍到基本记下）
  ↓
从参考书里选1本对标书
```

### 关键思考

| 思考方向 | 问题 |
|----------|------|
| 好看的书 | 哪些书最好看？读者评论总体和自己想法一样吗？ |
| 方向共性 | 最好看的书大概怎么写？有哪些相似性？ |
| 不好看的书 | 为什么不好看？成绩如何？是自己不是受众还是写得不好？ |

### 注意事项

- 不能上来就想靠标签或金手指类型筛选对标书，必须先广泛阅读
- 有些具体方向是互斥的，需要做选择
- 有经验的作者可以一眼判断方向，新人必须走完整流程
- 不被选做参考书的不再看，精力集中在参考书上

## 融书方法论

### 核心思路

找3本同题材成功书，认真看完并各写约2万字故事梗概，然后融合成新故事。

### 操作步骤

1. 找3本同平台百万在读且自己能看进去的书
2. 认真读完，给每本书写详细故事梗概（含读者评论分析）
3. 三层理解递进：
   - "人家讲了这么一个故事，所以有人看"
   - "人家用这种方式讲了这么一个故事，所以有人看"
   - "原来这种故事要如此去讲，才让读者更爱看"
4. 用掌握的理论框架（七步法/三翻四震等）把三本书融成新故事
5. 拿融合后的故事开书

### 检验方法

- 先把想写的东西用200字总结出来
- 看看是不是好故事、有没有意思、能不能吸引人
- 这是一切的起点：如果200字总结不吸引人，写出来也不会吸引人

### 注意

- 融合后的故事比大部分人自己想出来的更靠谱
- 有收入和正反馈后，再逐步把理论落实到书里
