# 人物设计与分析

## 第一部分: 人物设计（创作视角）

---

## 主角卡

```
姓名：
性别：
角色定位：（一句话说清主角在故事中的功能）
身份标签：（如：废柴大学生、前特种兵、落魄皇子）
外貌特征：（3-5 个关键词，不要写成通缉令，要有记忆点）
  例：瘦高、总穿旧夹克、左手有疤、眼神懒散
性格关键词：（3-5 个，必须有矛盾面）
  例：嘴毒心软、看似冷漠实则护短、冲动但有底线
核心目标：（全书终点想达成什么）
核心动机：（为什么一定要达成这个目标——情感驱动，不是理性驱动）
致命弱点：（让角色犯错的性格缺陷，不是能力缺陷）
口头禅/标志动作：（让读者秒认的标签）
```

**设计要点**：
- 动机必须是情感层面的（"为母亲复仇"优于"要成为最强"）
- 弱点必须会在关键情节导致主角犯错
- 外貌和口头禅是读者记忆锚点，宁可有也不要空

---

## 配角卡

```
姓名：
性别：
角色功能：（在故事中承担什么功能——导师/盟友/情报源/牺牲品/镜像对照）
与主角关系：
核心特质：（1-2 个关键词即可）
标志性特征：（一句话让读者记住）
退场方式：（何时/如何退出故事）
```

**设计要点**：
- 每个配角必须有明确功能，没有功能的角色不要出场
- 配角退场要主动，不能写着写着忘了
- 同一场景配角不超过 3 个有台词

---

## 反派层级

### 小反派（1-5 章）
```
功能：单个小弧线的障碍
设计：1-2 个鲜明特征即可
  例：嚣张、贪财、欺软怕硬
出场：快速建立冲突感
退场：被打败或被揭穿，干脆利落
```

### 中等反派（10-30 章）
```
功能：一卷的主要对手
设计：有动机、有手段、有逼格
  - 动机：为什么跟主角作对（不能是"纯粹的坏"）
  - 手段：靠什么威胁主角（武力/权谋/资源）
  - 逼格：至少赢主角一次，让读者恨得牙痒
退场：被主角正面击败，要有爽感
```

### 大弧Boss（一卷或数卷）
```
功能：代表一个阶段的核心矛盾
设计：完整的人物弧线
  - 有自己的目标和逻辑（从他的角度看他是"对的"）
  - 与主角有理念冲突（不只是利益冲突）
  - 至少有一场让主角陷入绝境的对决
  - 有一个让读者"恨不起来"的侧面
退场：有仪式感的终战 + 有余味的落幕
```

### 最终Boss（全书）
```
功能：全书核心矛盾的具象化
设计：从第一章就有伏笔
  - 与主角的目标/动机直接对立
  - 代表故事主题的反面
  - 实力碾压主角，主角必须蜕变才能赢
  - 有信念，不是纯粹的疯子/怪物
退场：与主角的终极对决 + 主题升华
```

**反派设计铁律**：
- 反派的智商/实力决定主角的含金量
- 弱反派 = 弱主角 = 无聊故事
- 反派的行为必须有内在逻辑（从他的视角说得通）
- 不要让反派降智来给主角送赢

### 反派建立四要素

1. **实力展示**：出场就要展示实力或手段，让读者记住"这人不好惹"
2. **动机可信**：从反派视角看，他的行为必须说得通，不是"纯粹的坏"
3. **对主角构成真实威胁**：至少赢主角一次，让读者恨得牙痒
4. **揭晓终极意图的时机**：反派的真实目的不要太早暴露，留到关键反转点

## 人物关系类型

| 关系类型 | 定义 | 功能 | 示例 |
|---------|------|------|------|
| 冲突型 | 双方利益/理念对立 | 制造张力，推动情节 | 宿敌、竞争对手 |
| 联盟型 | 双方有共同目标 | 提供助力，制造羁绊 | 战友、师徒 |
| 亲密型 | 情感纽带连接 | 制造软肋，提供情感支点 | 恋人、家人、兄弟 |
| 权威型 | 上下级/支配关系 | 制造压力，限制主角行动 | 师父、老板、监管者 |

**关系设计原则**：
- 每个重要关系至少要经历一次考验（背叛/牺牲/误解）
- 关系要有变化弧线（敌人变盟友、盟友变对手）
- 避免所有关系都是"你好我好"的铁板一块
- 关系的功能要服务于情节，不能只为甜/虐而存在

---

## 人设关联分层（强/中/弱关联）

角色设定按与剧情的关联强度分三层，确保每个角色都在故事中有明确的功能定位。

### 分层定义

| 层级 | 定义 | 功能 | 数量要求 |
|------|------|------|---------|
| 强关联 | 直接影响剧情走向和核心梗装逼爽点的设定 | 推动或引发人物碰撞、剧情推进 | 每个角色至少 3 个 |
| 中关联 | 配角的强关联设定，不能抢主角风头 | 为配角提供功能性和辨识度 | 适量 |
| 弱关联 | 丰富人物厚度的个人偏好设定 | 增加角色真实感和记忆点 | 不限 |

### 操作要点

- 主角的实力、钱财、人脉、背景等影响剧情走向的属性是强关联
- 角色喜欢吃蛋糕、爱好看美女、喜欢刀剑等属于弱关联，丰富人设用
- 同人角色天然有强关联（原著剧情和关系），原创角色需要主动设计
- 强关联是延伸剧情的核心，弱关联不能喧宾夺主

## 动机链

每个重要角色的核心动机用动机链描述：

```
起因（Cause）    → 角色经历了什么
意图（Intent）   → 角色想做什么
约束（Constraint）→ 什么在阻碍他
风险（Risk）     → 失败的代价是什么
```

**示例**：
```
起因：母亲被仇家杀害
意图：找到凶手并复仇
约束：凶手是当朝权臣，主角只是平民
风险：复仇失败会被灭门，复仇成功会失去所有现有生活
```

**动机链设计要点**：
- 起因必须具体（"被欺负"不够，"在众目睽睽下被打耳光"才行）
- 约束必须有力（否则主角一路碾压没有张力）
- 风险必须真实（读者要相信主角真的可能失去重要的东西）
- 动机可以随剧情演变，但不能说变就变

## 三层标签反差人设法

用三层标签构建角色，层与层之间的反差就是角色的立体感。

### 三层结构

| 层级 | 名称 | 含义 | 示例 |
|------|------|------|------|
| 第一层 | 身份标签 | 外界看到的身份 | 豪门弃妇 |
| 第二层 | 表现标签 | 角色展现给外界的行为 | 隐忍不发、逆来顺受 |
| 第三层 | 内核标签 | 角色真正的内心 | 冷静、有计划、步步为营 |

### 反差设计原则

- 三层标签之间必须有反差，反差 = 立体感
- 身份标签和表现标签可以相似（强化刻板印象），但内核标签必须反转
- 反差越大，角色的"亮牌时刻"越震撼
- 每个重要角色都应有至少一层反差

### 行为对照写法

用行为对比来体现反差，而非直接描述：

```
身份标签：豪门弃妇
表现行为：被骂不还口，被赶不反抗
内核行为：悄悄录音、收集证据、提前转移财产
```

## 角色塑造核心原则

### 主角行为三必须

1. **必须可理解**：读者必须理解主角为什么这么做
2. **必须可共鸣**：主角的思考读者必须能代入
3. **必须可接受**：主角的三观读者必须能接受

### 展示优于告知

用行动/选择展现性格，而非描述：
- 角色的目的 → 通过行为展示，不是旁白解释
- 角色的行动 → 具体的动作和决定，不是形容词堆砌
- 角色的态度 → 通过对话和反应体现，不是心理独白

### 角色不能太完美的信号

- 主角从不犯错 → 加致命弱点
- 主角没有成长空间 → 改起点设定
- 主角赢了没意义 → 检查反派强度

### 贴着人去写

行为、语言、思维都要围绕人格展开，不要为了剧情需要让角色做出不符合人设的行为。

---

## 反派塑造核心原则

经典反派人物一定有完整世界观和逻辑自洽的三观。坏人有自己的灵魂——有自己完整的三观和逻辑。
- 《大时代》丁蟹：彻头彻尾的疯子和坏人，但三观非常完整，有自己一套逻辑，心理状态非常自洽
- 同样的外部事件发生在不同人身上产生不同变化——外部事件是外因，内心世界是内因，变化是外因与内因共振的结果
- 反派塑造不能只有外部事件（外因），缺乏内在三观和内在逻辑

## 凸显人设的方法

- 抓亮点、主打事件推进、反差碰撞
- 通过行动展现人设（如《塔尔萨之王》开篇：主角出狱后不卑不亢、争取利益、展现老练）
- 反差拉张力会筛读者，要注意尺度

## 高情商/聪明人桥段写法

- 高情商在网文里通过具体剧情中的表现来体现
- 察言观色、临场应变、控场接话
- 最好的素材来自真实案例改编
- 拉片《红楼梦》等经典作品学习高情商桥段

## 人物扣的使用

用配角做"任务基地"——一个人物同时承载多个短期和长期期待：
- 主角每次解决事件装完逼后去找该人物开始新一轮装逼
- 每个剧情单元结束后利用同一人物展开新剧情
- 人物下线时带来更大的好处，用"歪打误撞收获更多"转变读者"损失厌恶"心理

## 配角人设逻辑

- "正戏反写"——看似正面的行为背后有反派动机
- 配角情感保护壳：看似冷漠无情，实则有深层情感驱动
- 避免崩人设：角色的行为选择必须符合已建立的人设逻辑
- 新手先从题材的"标签人物"直接用，成熟题材里的人设角色都是经受过市场考验的

---

## 配角功能化设计

### 核心原则
- 网文中写角色要像流水线的机械臂：功能化、简单化
- 配角的能力和特质必须和剧情紧密相关
- 配角是为了展现主角而写的：配角的能力为主角所用，配角脱离困境靠主角帮助

### 复合型配角
- 主角的白手套型配角：替主角说不适合说的话、发不适合发的狠
- 需要惊艳出场、绝对支持、绝对维护
- 比主角光环弱一点，但不能抢主角风头

### 立体人物 vs 扁平人物
- 不是所有角色都要立体化，要看功能性
- 立体人物鲜活但耗费精力，且某些角色（恐怖反派、高手）不适合过于立体
- 扁平化角色保持神秘性，反而更恐怖/更强大
- "洗白弱三分，黑化强三倍"：高手被立体化（底细被知道）就变弱了
- 不要把所有角色立体化，你需要什么工具就设计什么工具

### 人设反差技巧
- 表面性格 vs 内在性格的反差
- 外表形象 vs 表面性格的反差
- 外表形象 vs 内在性格的反差
- 身份 vs 生活方式的反差
- 身份 vs 爱好特长的反差
- 身份 vs 内在性格的反差

### 向人物深化
- 当前市场趋势是向人物深化，读者排斥复杂故事
- 在简单的、读者熟悉的故事中找到新的热闹
- 通过人物互动产生戏剧性，而不是把剧情复杂化

## 三层标签行为反差进阶

三层标签法的高级用法：当关系进入第三层（亲密关系）后，角色在第一、二层关系中能做出的行为反而做不出来了。

| 关系阶段 | 可做行为 | 不可做行为 |
|----------|---------|-----------|
| 身份标签期（陌生） | 调戏、冷漠、保持距离 | — |
| 表现标签期（熟悉） | 继续之前的行为模式 | — |
| 内核标签期（亲密） | 展露真实自我 | 之前的调戏/冷漠反而做不出来了 |

**写作要点**：
- 行为反差本身也是人设的一部分
- 例如：身份标签期爱调戏主角的辣妹，进入亲密关系后反而不敢表白——行为退化 = 感情深化
- 又如：反派阵营漠视生命的师傅，把主角当家人后反而下不去手——关系变化导致行为失能

## 人设的核心认知

**人设 ≠ 人物设定表**：
- 人物设定表 = 身高、外貌、家庭背景等信息堆砌（外在信息）
- 人设 = 围绕角色"灵魂/特殊人格"展开的功能性设计
- 人设的使命：服务戏剧张力与情绪拉扯，让角色在剧情中自动生长，以人格驱动行为，而非被剧情硬推

---

## 反派性格确立四步法

| 步骤 | 要点 | 写作要点 |
|------|------|---------|
| 1. 反派也有梦想 | 在反派眼中他是自己故事的主人公 | 必须让读者理解反派为什么这么做，不要求认同但要求理解 |
| 2. 挖掘创伤性过去 | 人物由过往塑造，反派由创伤步入歧途 | 谁抚养他？谁爱他？谁伤害他？他避免什么痛苦？ |
| 3. 磨练性格缺陷 | 遭遇逆境时反派不调动积极特质，而是磨练缺陷 | 反派的"优势"本身就是他的致命缺陷 |
| 4. 反派是主角的镜子 | 反派的长处反映主角的弱点 | 两者完全相反 = 每次相遇必然产生摩擦和冲突 |

**案例**：《凡人修仙传》韩立的谨慎 vs 宗门大派少主的猖狂，无论鬼灵门少主、乌丑、温天仁都一致对立。

## 展示人物特质的两种方法

### 1. 敏感性——触碰弱点
- 触碰人物的敏感领域来揭示性格缺陷
- 描述他对外在威胁的反应方式：惊慌失措？匆忙逃跑？转移注意力？
- 不同反应 = 不同性格

### 2. 逃避——展现回避行为
- 有性格缺陷的人物会主动避开某些触发因素（状况、人物、情绪、地点、颜色、气味）
- 展现角色如何回避这些触发因素 = 向读者暗示其弱点和过去的伤痕
- 角色的"不做什么"比"做了什么"更能揭示深层性格

### 3. 尽早展示吸引人特质
- 如果角色有负面特质，必须在第一时间揭示至少一个正面特质
- 读者先喜欢上角色，才能容忍其缺陷
- 顺序：先展示吸引人的特质 → 再逐步揭示缺陷

## 四大性格属性体系

从四大类别中至少各取一项主要特质来丰满主人公个性：

| 属性类别 | 定义 | 示例特质 | 设计要点 |
|---------|------|---------|---------|
| 道德属性 | 是非信念 | 仁慈、慷慨、荣誉、正派 | 影响其他性格的形成，如"正派"与"轻浮"矛盾 |
| 成就属性 | 推动达成目标 | 绩密、可靠、足智多谋、有条理 | 与道德属性一致，服务目标追求 |
| 互动属性 | 与人交流方式 | 耐心、谦恭、幽默、社会认知 | 数量最多的积极特质，受喜恶影响 |
| 身份属性 | 身份认同感 | 创新、怪异、超自然信仰、内省 | 定义人物个性的基本组成部分 |

**选择技巧**：以其中一个特质为主要特质，其他为辅。如赫敏的主要特质是"聪慧"，它直接影响其他行为、态度和抉择。

## 主角调子起太高的危害与化解

### 危害
- 失去成长空间：能力/地位/资源触达天花板，无法设计合理的升级线
- 剧情缺乏张力：陷入"砍瓜切菜"的重复循环，反派出场就被秒杀
- 读者失去追更动力："见证成长"是核心驱动力之一

### 化解方法
- 压势不压人：可以压低气势和期待，但不打压主角本身的能力和尊严
- 留出维度差距：即使某方面很强，总有其他维度是短板
- 超额收获法：看似收获的资源很大，经主角盘算后发现收获更大——正向升级，不要写致命陷阱

---

## 反派渲染分量的关键

- 反派分量不够 → 故事吸引力下滑
- 核心原则：**障碍从反派人设上来，而不是故意给主角制造障碍**
- 明确反派想要什么 + 为什么迫切想要 → 反派自然无所不用其极 → 主角自然陷入巨大麻烦
- 解决这种麻烦才能爆发情绪
- 反派渲染公式：明确动机 → 动机驱动行为 → 行为形成障碍 → 障碍的强度来自反派本身的分量

## 配角攻略缓冲区

### 概念
配角对主角态度的变化过程 = 主角攻略配角的过程，伴随巨大的期待感和爽点。

### 缓冲区类型
线上线下、背后议论、异地相处、地位差距、亲密度差距、信任程度、信息差等。

### 操作方法
1. 始终保持缓冲区存在
2. 在卷纲中挑出事件拐点（5~7个）
3. 在每个拐点处标注配角状态和对主角的态度变化
4. 每次攻略到关键点位时，配角的态度变化必须写清楚
5. 态度变化本身就产生情绪波动和期待感

### 注意
- 配角不能像NPC一样站着等主角触发
- 配角要有自己的行动，由配角观点引出事件
- 正面角色也一样：和主角立场相同的人也应有自己的行动和动机

---

## 人推事件 vs 事件推人

| 驱动方式 | 特点 | 效果 |
|---------|------|------|
| 人推事件 | 情节是人物性格/动机/选择的自然结果 | 深化人物弧光，角色更立体，增强代入感 |
| 事件推人 | 外部事件打破人物平衡 | 迫使暴露真实自我，但最终走向仍由人物选择决定 |

**核心原则**：
- 小说本质是"人如何应对世界"，而非"世界如何碾压人"
- 外部事件（战争/灾难/巧合）的作用是打破平衡状态
- 最终决定走向的仍是人物在失衡状态下的选择
- 卡文时：剖析笔下人物，从人物动机找方向和灵感

**案例**：《凡人修仙传》韩立加入七玄门看似事件推人，但墨大夫的存在说明是韩立的主动选择在驱动——人推着事件走，后续才是应变。

---

## 主角开篇立人设的关键

- 人设通过选择建立，开篇必须让主角做出选择
- 违背常理的现象：很多热门网文主角开篇用脑子解决问题，而非用金手指
- 智斗型开局比暴力型开局更容易立人设
- 读者更容易代入"聪明但普通"的主角，而非"上来就无敌"的主角

---

## 反派视角拉期待技巧

- 切到反派视角展示密谋计划 → 读者知道反派计划而主角不知道
- 产生信息差 → 读者的期待变成"主角如何反将一军"
- 注意：这种手法高手看来一眼看穿，不要过度使用
- 核心功能：深化反派人设 + 拉期待（读者期待看反派被打脸）
- 风险：过度使用会变成"谜语人"，降低悬念质量

## 感情流人设核心法

感情流中剧情为人设服务的核心方法论。

### 核心原则

- 感情流里剧情为人设服务：每次剧情都要丰满人物或推动感情发展，否则就是无效剧情
- 俗套剧情配上丰满人设也能脱离套路感，因为同一框架下人物自有独属风味
- 两个主角都必须有闪光点和独立变化路线，不能一个人出彩另一个人黯淡

### 人设核心构建法

1. **确定人设核心**：最初出现在脑海里的角色特征（如"想登上皇位的皇子""病弱皇子""破碎的人"）
2. **围绕核心发散**：为什么有这个核心？过去经历？环境影响？性格成因？
3. **延伸剧情线**：核心自带的剧情必须写（有目标→目标线；有伤痛→治愈线）

### 高光不等于装逼

所有让读者对角色印象深刻的时刻都是高光，包括痛彻心扉的时刻。

### 生命不止恋爱

角色心里希望恋爱，但生命里不能只有恋爱——只有恋爱的角色立不住、很单薄。

---

## 男频与女频爱情线底层逻辑差异

### 男频爱情线逻辑

- 核心是"外在因素展示"：主角和对象在一起体现的是两人的外在因素多优秀
- 三种核心逻辑："她要是我的该多好" → "这么优秀的人是我的了" → "这么优秀的人都喜欢我，我更优秀"
- 围观者想法应该是"能得到这么优秀的女人，我好羡慕"，而非"他们的恋爱好甜"
- 女主最好有多个女性角色喜欢主角，越多说明主角越优秀
- 爱情线对象本质上是"奖杯"——一切动机的根本目的是胜利

### 女频爱情线逻辑

- 核心是"连接"：两人之间那种唯一、坚不可摧、最优先的情感连接
- 感情越浓烈，越要求"你不是因为我的外貌财富才华才爱我，而是因为我是我"
- 连接形成后应逐渐舍去外在因素影响，保持纯洁性
- 双方最好是初恋+双洁——保障连接的纯洁性和唯一性
- 围观者想法应该是"他们的恋爱好甜，我好羡慕"
- 最好连连接的形成本身也减少外在因素影响

### 操作要点

- 写男频感情线时，围绕"展示优秀"设计事件
- 写女频感情线时，围绕"深化连接"设计事件
- 两者不可混淆，否则读者会觉得"不对味"

---

## 穿书/穿游戏角色选择法则

### 核心要求

- 叙事主角必须对原世界有相当熟悉度（这是该写法最核心的期待感来源）
- 确定穿越进熟悉的世界，一两句话交代清楚，不能超过一章还不知道身处何方
- 不是所有角色都能做穿越对象——必须选择穿越后有强戏剧性或强矛盾的角色（如穿越成恶毒女配）

### 信息差设计

- 在原世界基础上适当设计信息差，为叙事主角提供探索空间
- 可以用原世界知识卡bug获取超额收益

## 修罗场收场技巧

修罗场本质是一种矛盾，需要控制对抗烈度，不能让爱情线对象之间出现不可调和的矛盾。

### 收场方法

1. **关联回事业线**：让爱情线对象们把争风吃醋转化为"比谁对主角事业线贡献更大"，互相攀比为主角出力
2. **插入事业线突发事件**：当对抗进入白热化无法收场时，用一个事业线突发事件让所有人从内斗切换成一致对外，卸掉原势头
3. **一笔带过**（《斗破苍穹》式）：最多两人互相看不惯、说两句嘲讽话就过去，非常克制

---

## 人设改变的双向翻转法

爱情线中关系发生变化时的人设调整方法。

### 核心思路

- 最好的改变是两个人都改变，且改变后恰好与之前两人之间的对应关系相反
- 改变不是随便改变，而是基于原人设基础上的部分改变，和原人设有密切联系

### 注意事项

- 如果人设变化发生在爱隔山海阶段之后（尤其是面对最大阻碍之后），不要再设计新的矛盾，只要发糖
- 真正的阻碍已在前面解决，此时读者最迫切的渴望是多吃糖
- 发糖要用实在的CP行为和悉心照顾表达爱意，不是"我爱你"式的工业糖精

## 傻白甜角色塑造避坑

### 六个必避之坑

1. 女主犯错不能是故意的，不能明知不能做偏要做
2. 女主犯错不能违反道德——傻白甜最重要的是天真小孩子般的高道德感
3. 不能站在道德高地指责他人（道德婊行为），自己却做不好
4. 不能因为道德婊行为和队友发生矛盾后还让队友认错赞美她
5. 好的傻白甜是：有更高道德标准 → 所作所为符合这个标准 → 为此显得与众不同和有点傻气
6. 可以写成长线：坐到被她指责的人的位置上后，发现原来那个做法已是最好的选择

### 反向用法

如果塑造傻白甜反派，就把以上六点全加在她身上，让读者一见就厌恶——本质上是戏剧三角的思路。

## 以身入局：作者与角色共情四步法

代入感的核心不是技巧，而是作者与角色共情。具体步骤：

### 第一步：放下自己

不能让作者的思想性格决断影响角色的思想性格决断，否则陷入自嗨，脱离读者需求。

网文是商品：放下自己，把读者当客户，把主角当客户。

### 第二步：保持清醒

在代入角色前保持绝对理性，问自己三个问题：
1. 这段剧情我想表达什么情绪，有什么目的？
2. 人物设定展示和要表达的情感是否符合当前情绪需求？
3. 理性看待每个角色使命——既能共情角色，又能瞬间脱离不受影响

**反面案例**：作者沉浸于主角的悲伤情绪无法自拔，后续剧情全部在悲伤中蔓延，情绪核心崩塌。

### 第三步：代入角色

- 了解角色的个人经历、思想性格、喜好特点、言行举止、目标动机
- 越重要的角色了解越全面，尤其是主角
- 在具体剧情场景中，与角色融为一体，感受所思所想
- 满足剧情需要、不脱离人设、还原角色情绪内核

### 第四步：共情演绎

- 确定情绪核心与剧情/人物展示的情绪是否一致
- 体会角色情感需求，按人设还原再现真实情绪
- 与角色共情后，笔下每段剧情、每个人物都充满情绪

**两种实践路径**：
- 路径A：先构思剧情→代入角色共情→演绎
- 路径B：先代入角色共情→根据情绪核心构思剧情

共情后还需用代入感技巧（读者共情、氛围感、主角视角、画面感）再打磨一遍。

## 复合型配角设计：白手套角色

### 定义

主角的好兄弟/铁杆盟友，暴脾气、绝对信任、绝对支持，是主角的"白手套"。

### 功能

| 功能 | 说明 |
|------|------|
| 替主角说话 | 凡是主角不适合说的话，他来说 |
| 替主角发狠 | 凡是主角不应该发的狠，他来发 |
| 绝对维护 | 对侵犯主角利益的人坚决反击，斩草除根 |
| 粗中有细 | 将反派密谋录音再播放，取得大众支持 |
| 适时搞笑 | 不断输出正反馈，缓解紧张情绪 |

### 设计原则

- 光环比主角弱一点，不抢主角风头
- 惊艳的出场（用悬念方式引入）
- 对主角的质疑第一时间反驳
- 反嘲讽能力要强

## 职场模型：理解一切网文关系的方法论

### 核心认知

网文本质：在一个"职场"爽透，然后再到另一个"职场"爽。

### 职场模型的共性

| 维度 | 说明 |
|------|------|
| 层级 | 任何环境中都有上下级关系（学校、宗门、公司、军队） |
| 互助 | 同级之间的合作和互利 |
| 互害 | 同级之间的竞争和倾轧 |

### 职场模型的差异

| 维度 | 说明 |
|------|------|
| 合作方式 | 不同环境合作模式不同（师徒/队友/同门） |
| 资源类型 | 不同环境争夺的资源不同（修炼资源/客户/功劳） |
| 个体能力 | 主角在其中的独特优势（金手指/信息差/特殊技能） |

### 操作方法

1. 为主角设计一个"职场环境"（学校/武馆/宗门/公司/军队）
2. 在环境中设置层级关系（上级/同级/下级/对手）
3. 利用层级关系自然产生矛盾和冲突
4. 主角在这个环境中爽透后，换到更大的"职场"

**适用范围**：几乎所有网文题材都能用职场模型理解——诡秘之主（探案职场）、斗破苍穹（家族/学院职场）、全职高手（电竞职场）。

## 人物塑造与人物真相

### 核心概念

人物构成分为两层：

| 层面 | 定义 | 示例 |
|------|------|------|
| 人物塑造 | 外在显性特征 | 外表青春可爱的萝莉 |
| 人物真相 | 内在隐蔽的内心 | 实际上是冷血杀手 |

**人物张力来源**：外表和内在的反差。读者根据人物塑造推测角色，当人物真相与推测相反时产生强烈戏剧性。

### 三个层级

1. **人物塑造（显型）**：读者第一眼看到的特质
2. **人物真相（隐性）**：角色内在的真实面貌
3. **价值取向（内隐）**：在关键选择时起决定作用的价值观

**价值取向决定剧情走向**：岳不群表面君子剑（人物塑造），内里虚伪野心家（人物真相），在道义与野心间选择野心（价值取向），导致林家灭门。

**操作要点**：
- 价值取向不必一开始就揭示，在关键抉择时自然暴露
- 人物真相的揭示越晚，戏剧张力越强
- 人物塑造和人物真相的反差越大，角色越有记忆点

## 向人设深化法

### 核心认知

当前市场趋势明确向人设深化走，向故事深化（老白文思路）越来越被新读者排斥。

**两种深化方向对比**：

| 方向 | 做法 | 优势 | 劣势 |
|------|------|------|------|
| 向故事深化 | 将简单剧情复杂化（多方谋划、层层反转） | 上限理论上高 | 耗时耗力、读者未必买账 |
| 向人设深化 | 在简单故事中通过人物戏剧性找乐子 | 省力、读者买账 | 需要人设功底 |

### 向人设深化的操作

**核心**：在目标读者熟悉的故事中，通过人物身上找读者感兴趣的戏剧性（乐子）。

**操作步骤**：
1. 拿一个简单甚至老套的故事结构
2. 在人物身上加"戏剧性"——能让目标读者群体找到乐子的设定
3. 通过几个人物的互动循环产生持续剧情

**案例示范**（师叔逼迫女弟子勾引男主的简单情节）：
- 女弟子：恋爱脑，为恋人宁死不屈
- 恋人（青梅竹马）：表面威逼女弟子是为了前程，实际有绿帽癖在暗爽
- 读者乐子：看着恋人表面痛心疾首实际暗爽的反差

**循环写法**（一章2000字可写5章以上）：
- 第1章：主角装逼 → 师叔更迫切
- 第2章：师叔逼女生 → 恋人帮着威逼利诱
- 第3章：女生靠近主角 → 恋人在暗地暗爽
- 第4章：女生回去哭诉 → 恋人假意安慰一边暗爽
- 第5章：复命领赏 → 女生发现恋人对奖赏没那么高兴 → 疑心被恋爱脑压下

### 戏剧性不放主角也可以

戏剧性可以放在任何角色身上，包括可攻略对象和叙事主角：
- 男主警察+运气设定：早餐对面坐着的一定是当前犯人 → 和女主相遇回到"女人你好特别"
- 把同样设定放女主身上：女警员凭运气破案 → 局长男主主动约早餐 → 女主拒绝（怕运气让上司进局子）→ "女人你好特别"

### 安全感与核心情绪把控

- 不同题材的读者对安全感需求不同
- 极道流等紧张感题材：安全感拉爆 = 危机感消失 = 核心情绪偏移
- 给主角找过强的靠山是常见错误——直接消解了紧迫感
- 精力有限时，情节始终是第一位的，世界观只是加分项

## 声望落差理论：门当户对的多维理解

### 核心概念

同一个角色在不同人眼中的"声望"是动态变化的，不是恒定值。

以男主拥有ABCD四样东西（家境/学历/工作/性格）为例：

| 视角 | 看重什么 | 对男主的评价 |
|------|----------|-------------|
| 世俗视角 | 家境对等，抗风险能力 | 综合条件匹配度 |
| 男主自身 | 赚钱养家+感情 | 相对门当户对 |
| 女主视角 | 感情+专一+爱 | 只要在乎的就门当户对 |
| 富二代 | 外貌家世匹配度 | 我才更门当户对 |
| 路人 | 综合条件对比 | 女主该嫁富二代 |

**操作要点**：
- 不同人对同一个角色的评价差异 = 天然的矛盾冲突来源
- 恋爱文的核心爽点之一：不同维度的评价差
- 不要太考究理论，大多数时候靠本能写："我觉得这样写很爽，于是就写了"
- 写完再拿理论印证可以，但先拿理论设计桥段容易吃力不讨好

## 压势不压人：矛盾的正确来源

### 错误做法

让反派莫名其妙针对主角、辱骂主角、打压主角，背后逻辑站不住脚。

### 正确做法：压势不压人

反派有自己的目的（如打压某地区修行者获取资源），他的行动客观上导致了主角的困境，但并非刻意针对主角本人。

**矛盾链条**：
1. 反派的目的 → 客观上压制了主角所在群体（压势）
2. 主角推翻反派 → 但底层群体依旧麻木
3. 主角改变路径（互利共赢）→ 但群体中有人利用善意
4. 新的冲突自然产生 → 循环继续

**核心**：所有人被自己想要的利益驱动，背后是三观不同，更深是成长环境，最深层是世界观。只要你想，一本书的矛盾冲突可以时时刻刻产生。

## 核心情绪与表达欲平衡

### 问题：表达欲导致核心情绪崩坏

作者陷入自嗨后偏离核心梗，违背读者需求，导致订阅腰斩、追读狂跌。

### 案例：《完美世界》核心情绪崩坏

- 前期精彩，下界几乎完美
- 上界开始核心情绪崩塌——为了独断万古的悲情，所有铺垫都服务于这一目的
- 读者感受：天下无敌了还苦哈哈一个人？啥也得不到？那练个鬼？
- 结果：月票榜直线下降

**修复方式**：独断万古可以，但主角要有收获。实力升华+亲人安全+减少悲剧——既保留悲情豪迈，又不违背读者需求。

### 案例：《十方武圣》天地独行

- 换地图时杀掉所有NPC，老婆死绝
- 核心情绪严重崩坏，剧情严重割裂
- 修复：换地图可以，但不杀原地图的人；老婆病重需救治→新地图找解药→期待感来了

### 核心原则

- 需求的内核不变，变的只是形式
- 写完一段剧情后，站在读者角度审视是否偏离核心梗
- 判断不了时，记住：主角牛逼了必须有所得，不能光付出没回报
- 表达欲旺盛时最容易偏离——要主动检查

## 第二部分: 人物分析（拆文视角）

---

## 一、人物元素提取

### 提取模板

```
角色名：
性别：
角色定位：主角 / 对手 / 盟友 / 导师 / 工具人 / 隐藏角色
身份标签：{社会身份，如「落魄世家子弟」「金牌律师」「村霸」}
外貌关键词：{3-5 个词，不是详细描写，是关键特征}
性格关键词：{3-5 个核心性格词}
核心目标：{这个角色最想达成什么}
深层动机：{为什么想达成这个目标（和目标不同）}
致命弱点：{这个角色的软肋/盲点}
口头禅/标志性动作：{}
```

### 提取要点

- **身份标签**比详细背景更重要。标签决定读者对这个角色的第一判断
- **核心目标**和**深层动机**要分开看。目标是他做什么，动机是他为什么做。好人物的目标和动机常有张力（表面追求 A，实际渴望 B）
- **致命弱点**是推动剧情的关键。没有弱点的角色是工具人，有弱点的角色才是人

### 角色功能分类

| 类型 | 功能 | 判断标准 |
|------|------|----------|
| 主角 | 读者代入 + 推动主线 | 视角最多、成长弧线最完整 |
| 对手 | 制造冲突 + 推动成长 | 和主角目标直接对立 |
| 盟友 | 辅助主角 + 提供情感价值 | 在关键时刻帮助主角 |
| 导师 | 传授能力 + 指引方向 | 比主角强、阶段性出场 |
| 工具人 | 推动某个情节 | 功能单一、出场后可退场 |
| 隐藏角色 | 制造反转/悬念 | 前期不露面但影响全局 |

## 二、人物关系映射

### 四种关系类型

#### 1. 冲突关系

```
角色 A ←→ 角色 B
冲突类型：利益冲突 / 价值观冲突 / 情感冲突 / 生存冲突
冲突强度：1-10
触发事件：{什么引起了冲突}
升级方式：{冲突如何一步步加剧}
```

分析要点：
- 冲突是否有升级路线（小摩擦 → 大对抗 → 生死战）
- 冲突的根源是否足够深（不是单纯的脸谱化对立）
- 冲突是否能推动双方的角色发展

#### 2. 结盟关系

```
角色 A ←→ 角色 B
结盟基础：利益一致 / 情感羁绊 / 共同敌人
信任程度：低 / 中 / 高 / 绝对信任
潜在裂痕：{什么可能导致联盟破裂}
```

分析要点：
- 结盟是否有利益基础（不是无缘无故的帮忙）
- 是否有潜在矛盾（盟友间可以有分歧，增加真实感）
- 结盟是否推动主线发展

#### 3. 亲密关系

```
角色 A ←→ 角色 B
关系类型：恋人 / 家人 / 师徒 / 知己
情感深度：表面 / 深层 / 灵魂级
关键场景：{定义这段关系的核心场景}
变化轨迹：{从陌生到亲密的过程，或从亲密到决裂}
```

分析要点：
- 亲密关系的建立是否有足够的铺垫和事件支撑
- 关系变化是否推动剧情（不只是情感线）
- 是否有独特的互动细节让这段关系区别于其他作品

#### 4. 权力关系

```
角色 A（上位）→ 角色 B（下位）
权力来源：身份 / 实力 / 资源 / 信息
权力动态：固定 / 翻转中 / 已翻转
```

分析要点：
- 权力关系是否有动态变化（权力翻转是大爽点来源）
- 权力差是否被合理利用（制造压迫感/释放爽感）
- 翻转是否有铺垫（不能无理由翻转）

### 关系图绘制

对全书或一卷绘制人物关系网：

```
中心：主角
第一圈：与主角直接互动的角色（标注关系类型和强度）
第二圈：通过第一圈角色间接影响主角的角色
标注：权力方向（→）、冲突强度（粗线）、亲密程度（虚线）
```

## 三、动机链分析

### 动机链模型

```
起因（Cause）
  → 触发事件：{什么事件触发了角色的行动}
  → 角色处境：{事件发生时角色的状态}

意图（Intent）
  → 表面意图：{角色说自己要做什么}
  → 真实意图：{角色实际想达到什么}
  → 意图层级：{表层目标 → 中层需求 → 深层渴望}

约束（Constraint）
  → 外部约束：{客观条件限制（实力不够/没有资源/被人阻止）}
  → 内部约束：{角色自身的限制（性格弱点/道德底线/情感羁绊）}

风险（Risk）
  → 失败代价：{如果行动失败会失去什么}
  → 成功代价：{如果行动成功要牺牲什么}
  → 道德代价：{这个行动是否有道德争议}
```

### 动机链检查清单

- [ ] 每个重要行动是否有明确的起因（不是无缘无故）
- [ ] 表面意图和真实意图是否有区分（复杂的角色不会直说真实想法）
- [ ] 约束是否合理（不能让角色太容易达成目标）
- [ ] 风险是否足够（高风险才有关注度）
- [ ] 角色的关键抉择是否反映动机链（在约束和风险下的选择最能体现角色本质）

### 动机冲突类型

| 类型 | 定义 | 经典例子 |
|------|------|----------|
| 目标 vs 道德 | 想做的事和应该做的事冲突 | 复仇但不想伤及无辜 |
| 情感 vs 利益 | 感性和理性的拉扯 | 知道对方不可靠但放不下 |
| 短期 vs 长期 | 眼前利益和远大目标的矛盾 | 现在妥协保命 vs 坚持原则赌未来 |
| 自我 vs 他人 | 个人需求和他人期望的冲突 | 想过平静生活但背负家族使命 |

## 四、跨章节一致性检查

### 检查维度

#### 1. 性格一致性

```
检查项：角色在不同场景下的行为是否符合同一性格设定
方法：
  - 列出角色的核心性格词
  - 在 5 个关键场景中标记角色的行为
  - 检查行为是否和性格词一致
注意：
  - 角色可以成长和变化，但变化要有铺垫
  - 压力下的行为最能体现真实性格
  - 「在 XX 情况下会做 YY」是比性格词更精准的描述
```

#### 2. 关系一致性

```
检查项：角色间的关系变化是否有迹可循
方法：
  - 按章节顺序列出 A-B 关系的关键变化点
  - 检查每个变化点是否有触发事件
  - 检查是否有突然变化但缺乏铺垫的情况
常见问题：
  - 关系突然变好/变差没有理由
  - 前面的矛盾后面突然不存在了
  - 亲密关系缺少建立过程
```

#### 3. 能力一致性

```
检查项：角色的实力/能力是否前后一致
方法：
  - 标记角色每次展示实力的场景
  - 对比同类型场景下角色的表现是否一致
常见问题：
  - 主角有时强有时弱，取决于剧情需要（战力崩坏）
  - 配角在有主角在场时莫名变弱
  - 设定的限制在后面被遗忘
```

#### 4. 信息一致性

```
检查项：角色知道什么、不知道什么是否前后一致
方法：
  - 列出关键信息的揭露时间点
  - 检查角色在某个时间点之前不应该知道某些信息
  - 检查角色是否利用了不应该知道的信息
常见问题：
  - 重生文中最常见：主角使用了前世记忆中没有的信息
  - 角色在 A 章不知道的事在 B 章突然知道了但没人告诉他
  - 悬疑文中：角色的行为暗示他知道真相但设定上他还不知道
```

### 一致性检查频率建议

| 检查时机 | 检查范围 | 重点关注 |
|----------|----------|----------|
| 每 10-20 章 | 最近章节的所有角色 | 性格 + 关系 |
| 每卷结束 | 本卷所有角色 + 跨卷主线角色 | 能力 + 信息 |
| 全书完稿 | 所有重要角色 | 全部四维度 |

### 检查输出格式

```
角色：{名字}
检查范围：第 X 章 ~ 第 Y 章

问题 1：{问题描述}
  - 位置：第 N 章
  - 前文设定：{之前怎么设定的}
  - 当前矛盾：{现在怎么写的}
  - 严重程度：高/中/低
  - 修复建议：{怎么改}

问题 2：...
```
